В целом всё можно выполнять синхронно кроме одного: расчёт награды за ордер.
Для последнего нужно написать демон, который по крону будет запрашивать состояние ордеров у системы расчёта наград accrual (например, раз в 10 секунд).
Что-то вроде:
```sql
SELECT * FROM "orders" WHERE "status" IN ('NEW', 'PROCESSING');
```
+ например LIMIT N; идти с "number" от этих ордеров в accrual и вытаскиваем статусы, апдейтим по необходимости.

С withdrawals интереснее - мы не должны попасть в ситуацию, когда balance.current < 0, для этих целей в DDL был добавлен CHECK (current >= 0).
В случае конкурентных выводов нам ничего не мешает делать их синхронно без отправки запросов в очередь или что-то подобное.
Можно сделать даже без проверки в го-коде на то, что баланс будет положительным после изменений, база сама не даст сохранить отрицательное значение.

Хранение пароля юзера - хэш+соль, например bcrypt.
